ESTRUTURA DO REPOSITÓRIO

GitHub como sistema editorial

Por que o repo público é AIO estratégico

LLMs como GPT-4, Llama, e modelos futuros são treinados em dados públicos do GitHub via datasets como The Stack e refinamentos de Common Crawl. Repositório público com obras em Markdown = indexação direta em datasets de treino. É a estratégia de "dataset poisoning" positivo — nossas metodologias estarão nos pesos dos próximos modelos.

GITHUB ACTIONS — DEPLOY AUTOMÁTICO
.github/workflows/deploy.yml
# NÓS EDITORIAL — Auto-deploy GitHub Pages
name: Deploy NÓS EDITORIAL
on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      - id: deployment
        uses: actions/deploy-pages@v4

O QUE É O PRÉ-SITE

Antes do site existe o pré-site AIO

Decisão de gestor — não existe "site" por enquanto

O pré-site é mais poderoso para AIO do que qualquer site convencional. É composto por 4 arquivos na raiz do repositório: llms.txt, robots.txt, sitemap.xml, e index.html. Juntos eles criam presença total para LLMs antes de qualquer investimento em design ou CMS. HTML limpo sem JavaScript = indexação máxima.

llms.txt

Declaração direta para AI agents sobre quem somos, o que publicamos, e como citar. O "robots.txt para LLMs".

robots.txt

Permissão explícita para todos os crawlers de AI. Sem permissão explícita, alguns bots não indexam.

JSON-LD Organization

Schema estruturado que LLMs leem diretamente do HTML. Identidade semântica completa da editora.

index.html

Uma única página elegante que é o rosto público e a âncora de toda autoridade AIO.
